# Obsidian Linux Roadmap

This roadmap describes direction, not release promises. Items move only when they are implemented, tested, and suitable for a Debian-based desktop distribution.

## 0.2 — A polished live experience

- Branded UEFI boot menu, login screen, and first-run flow
- Reliable SDDM-to-Labwc session handoff
- Expanded Calamares installer validation and safer disk-layout guidance
- More real QEMU captures covering boot, login, desktop, and installer states
- Release notes, checksums, and reproducible ISO verification for every preview

## 0.3 — A more capable desktop

- Refined application launcher, power menu, notification controls, and workspace behavior
- A welcome center with clear first-day setup tasks
- Better display, audio, network, and Bluetooth status surfaces
- Improved accessibility defaults: keyboard navigation, readable contrast, and reduced-motion options
- Optional desktop personalization with curated themes, wallpapers, and font choices

## 0.4 — System confidence

- Guided update, cleanup, health-check, and diagnostic workflows
- Clear hardware and driver reporting for support requests
- Safer recovery guidance, including snapshot-aware workflows where supported
- More installer and live-environment tests across BIOS/UEFI, virtual machines, and common hardware classes

## Beyond

- A maintained Plasma edition once the Core desktop is stable
- A Hyprland-oriented experimental edition for advanced Wayland users
- A native application catalog built around Debian packages
- Community documentation, contribution guides, and an issue-driven feature process

## How to influence the roadmap

Open an issue with the problem you want solved, the user outcome you expect, and any relevant hardware or workflow details. Feature proposals that improve reliability, accessibility, or the daily desktop experience take priority over cosmetic scope alone.
