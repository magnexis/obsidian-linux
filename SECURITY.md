# Security Policy

## Supported Scope

Obsidian Linux is in early development. Security-related reports are still welcome, especially for:

- build scripts
- packaging behavior
- installer configuration
- privilege boundaries around system installation

## Reporting a Vulnerability

Please do not open a public issue for a suspected security problem before maintainers have had a chance to review it.

For now, use a private maintainer contact channel or repository security advisory workflow when available.

## Response Expectations

- Reports should include affected files, reproduction details, and impact.
- If the issue is not yet verified, say so clearly.
- Fixes should avoid broad unrelated refactors.
