# Release Process

## Current Release Model

Obsidian Linux is pre-release software. The intended release flow is:

1. Build custom packages
2. Build the live ISO
3. Validate package and ISO outputs
4. Capture updated real screenshots from the running system
5. Prepare release notes and checksums
6. Publish the ISO artifact and changelog

## Requirement

Do not publish a release that advertises screenshots or features that were not verified from the current build.
